============================================
  수정이 화면에 안 보일 때 확인할 것
============================================

1. 어디서 보고 계신가요?
   - 로컬: 브라우저에서 http://localhost:포트/10trix/ 로 여시나요?
   - 배포: Vercel 주소로 보시나요? (푸시 후 1~2분 뒤에 반영됩니다)

2. 캐시 비우기 (가장 흔한 원인)
   - Windows: Ctrl + Shift + R (강력 새로고침)
   - 또는 Ctrl + F5
   - 또는 브라우저 설정에서 "캐시 비우기" 후 다시 접속

3. 수정된 버전인지 확인하는 방법
   - 페이지에서 마우스 오른쪽 클릭 → "페이지 소스 보기"
   - Ctrl+F 로 "10trix v202602" 검색 → 있으면 최신 HTML이 로드된 것
   - "custom.css?v=10trix202602" 검색 → 있으면 CSS 링크도 최신

4. 로컬 서버로 확인할 때
   - 반드시 "파일 더블클릭"이 아니라, 서버 실행 후
     http://localhost:xxxx/10trix/ 처럼 주소로 접속하세요.
   - 예: start-server.bat 또는 재시작-서버.bat 실행 후 브라우저에서 접속

5. Vercel 배포본을 보실 때
   - GitHub에 푸시한 뒤 1~2분 기다리기
   - Vercel 대시보드 → Deployments 에서 최신 배포가 "Ready"인지 확인
   - 브라우저에서 해당 Vercel URL로 강력 새로고침 (Ctrl+Shift+R)
